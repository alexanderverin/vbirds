VerinBirds — описание проекта

VerinBirds — CV-модуль овоскопирования яиц

Junior ML Contest ITMO (AI Talent Hub), категория AI-стартап. Описание проекта (≤3 стр.).

1. Проблема и целевая аудитория

Овоскопирование (просвечивание яйца лампой) — стандартный способ во время инкубации отличить оплодотворённое яйцо (fertile) от пустого (infertile) и замершего (dead), чтобы вовремя убрать нежизнеспособные и не терять место в инкубаторе. На перепелиной ферме это ручная, субъективная и утомительная операция: партии по 100-200+ яиц, оценка «на глаз», результат зависит от опыта и усталости оператора.

Целевая аудитория: небольшие и средние перепелиные (и куриные) хозяйства, где нет промышленных систем контроля инкубации. Ценность: ассистент, который по фото на просвете подсказывает класс яйца и попутно накапливает размеченный датасет фермы.

Конкуренты / состояние области. Автоматический candling для кур исследуется (точность ~85% в литературе). Для перепелов оптическое просвечивание почти не применяют: мелкий размер и пигментированная пятнистая скорлупа плохо пропускают свет — исследователи переходят на тепловизор/ИК (напр. термокамера + YOLO, MDPI Sensors 22(15):5820, 2022). Это ключевой факт, определяющий и наш научный результат, и roadmap.

2. Данные и разведочный анализ (EDA)

Обучающий источник — публичный датасет куриного овоскопирования (hlnkmb, 4275 фото, 3 класса), т.к. перепелиного датасета овоскопирования в открытом доступе не существует. Датасет идеально сбалансирован (1425/класс) и имеет готовый train/test сплит 80/20.

Главная находка EDA — утечка через артефакты сбора данных. Классы систематически различаются не биологией яйца, а условиями съёмки:

  • fertile-фото крупнее и светлее (медиана яркости 72 против 48 у infertile и 42 у dead); dead/infertile заранее сжаты до 416×416, fertile — из отдельного источника (часть кадров — веб-экспорт с вотермаркой).
  • Количественная проверка: дерево решений только по метаданным (размер, пропорции, яркость), без содержимого, даёт accuracy 0.648 при случайной 0.333; SVM по 3 числам (средний R/G/B) — 0.881.

Вывод, определивший весь дальнейший ход работы: высокую точность на этом датасете нельзя принимать за «навык» модели — она в значительной мере отражает источник фото.

3. Метод

Пайплайн вынесен в чистые модули (/ml), обёрнут в API (/backend), обучение — в облаке.

  • Препроцессинг: единый вход 224×224 с letterbox (сохранение пропорций + чёрный паддинг), чтобы убрать «размер/пропорции» как подсказку класса и не искажать яйца. Аугментации подобраны под специфику овоскопирования (flip, поворот ±15°, яркость/контраст ±20%); осознанно исключены сильный сдвиг цвета (краснота свечения — сигнал) и вертикальный флип (воздушная камера задаёт «верх»).
  • Baseline: классические признаки (нормированные цветовые гистограммы R/G/B + GLCM-текстура) → SVM.
  • CNN (transfer learning): ResNet18 (ImageNet) как замороженный экстрактор признаков + логистическая регрессия поверх 512-мерных эмбеддингов (linear probe). Обучение — в облаке (Colab), без обучения на CPU ноутбука; ресурсоёмкость минимальна (только forward-проход).

4. Результаты (честно)

Модель Test accuracy (куры) Структура ошибок
Baseline (цвет+текстура → SVM) 0.998 2 ошибки на 855, случайные
CNN linear probe (ResNet18 → логрегрессия) 0.993 fertile отделён идеально; путаются только infertile↔︎dead

Обе модели дают ~99% — но, как показано в §2, этот показатель насыщен артефактами источника, поэтому «победитель» по куриному тесту не выбирается. Мы намеренно не «улучшали» цифру постфактум и не выдаём её за качество распознавания. Небольшой содержательный плюс CNN: его ошибки осмысленнее (путаются два биологически похожих тёмных класса), что указывает на опору на структуру, а не только глобальный цвет.

Научный результат работы — не accuracy, а честная фиксация domain gap. Прямой перенос куриной модели на перепелов ограничен пигментацией скорлупы; это подтверждается литературой и объясняет, почему для перепелов нужен другой источник света (ИК/термо). Отсутствие перепелиного candling-датасета — объективное состояние области, а не пробел проекта.

5. Продукт (MVP)

  • API /predict (FastAPI, лёгкий образ без torch): фото → класс + confidence + честная пометка «обучено на курах, на перепелах не валидировано» прямо в ответе. Есть фильтр «это фото овоскопирования?» (эвристика, откалибрована: <1% ложных отклонений реальных яиц).
  • Telegram-бот сбора данных, задеплоен в Docker на VPS: съёмка яйца → модель подсказывает класс → подтверждение одним касанием; счётчик партии, отбраковка при раскатке, напоминания по датам овоскопирования и вылупления. ID яйца = карандашный номер (переживает раскатку) → даёт настоящий per-egg ground truth (реально вылупилось/нет), которого нет в публичных датасетах.
  • Разметка на лету решает ключевую проблему: на факте вылупления фермер уже не помнит визуальную картину каждого яйца.

6. Ограничения и Roadmap

Ограничения (заявлены честно): domain gap кура→перепел не измерен (нет данных); куриный тест артефактный; фильтр фото не отличает разбитое яйцо (проверяет стиль снимка, не целостность); API отдаёт baseline, CNN — исследовательский артефакт.

Roadmap: (1) сбор перепелиного датасета через бота (процесс запущен); (2) дообучение модели на перепелах, когда наберётся выборка; (3) детектор «яйцо/не-яйцо» и класс cracked; (4) эксперимент с ИК/термо-подсветкой для перепелов (направление, подтверждённое литературой).

7. Инженерия и применение ИИ

Стек: Python 3.11, FastAPI, scikit-learn/scikit-image, PyTorch (обучение в Colab), Docker + docker-compose (задеплоено на VPS), структурированный репозиторий с журналом инженерных решений. Тяжёлые ML-либы вынесены из рантайм-образа (образ ~721 МБ, без torch).

Применение ИИ в продукте: ИИ работает в цикле разметки (model-assisted labeling) — модель предлагает класс, оператор подтверждает/исправляет, и исправления формируют новый датасет фермы. Каждое инженерное решение и метрика зафиксированы по шагам в журнале решений проекта.
